# Instrumentation for Axum-todo

Generated by TelemetryOS for the agreed SLO contract (v1). This PR adds the
OpenTelemetry instrumentation needed to emit the SLIs selected below.

## SLIs implemented
- **Axum-todo — HTTP Service Availability** (`axum-todo-http-availability`) — target 99.95% over 30-day
- **Axum-todo — HTTP Response Time P95** (`axum-todo-http-latency-p95`) — target < 200ms over 1-hour
- **Axum-todo — HTTP Response Time P99** (`axum-todo-http-latency-p99`) — target < 750ms over 1-hour
- **Axum-todo — HTTP 5xx Error Rate** (`axum-todo-http-error-rate`) — target < 0.1% over 1-hour
- **Axum-todo — HTTP Worker Pool Saturation** (`axum-todo-http-saturation`) — target < 80% over 5-minute
- **Axum-todo — Authentication Failure Rate** (`axum-todo-http-auth-failure-rate`) — target < 2% over 1-hour
- **Axum-todo — HTTP Request Throughput** (`axum-todo-http-request-rate`) — target baseline ± 30% over 5-minute
- **Primary Flow — E2E Business Flow Success Rate** (`flow-primary-flow-e2e-flow-success`) — target 99.95% over 30-day
- **Primary Flow — E2E Business Flow Latency P95** (`flow-primary-flow-e2e-flow-latency`) — target < 500ms over 24-hour
- **Primary Flow — Flow Validation Failure Rate** (`flow-primary-flow-e2e-validation-failure-rate`) — target < 5% over 24-hour
- **Primary Flow — E2E Business Flow Throughput** (`flow-primary-flow-e2e-flow-throughput`) — target baseline ± 25% over 1-hour
- **Primary Flow — Flow Completion Freshness** (`flow-primary-flow-e2e-flow-freshness`) — target < 2 minutes over 1-hour
